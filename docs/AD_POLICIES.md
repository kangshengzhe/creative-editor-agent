# 广告平台政策与字符限制参考

> 本文档汇总本项目支持的三个广告平台（Google Ads、Meta/Facebook Ads、TikTok Ads）的**文案字符限制**与**关键内容政策**，作为文案生成与截断逻辑的依据。
>
> ⚠️ **重要提醒**：广告平台政策更新频繁，且常因国家/地区、行业（尤其博彩、金融、医疗）而异。本文档基于截至 **2025 年**的公开官方资料整理，**投放前必须以各平台后台和官方帮助中心的最新规则为准**。文中内容为合规改写，非逐字引用。

---

## 一、为什么字符限制如此重要

广告平台对每个文案字段有**硬性字符上限**。超限会导致两种后果：

1. **被系统拒登 / 截断**：超出部分被平台自动截掉，可能把句子切成半截，损害品牌形象。
2. **预览与实际显示不一致**：即使后台接受，feed 中也可能因空间不足而省略尾部。

因此本项目的原则是：**宁可文案短一些，也绝不能生成一句被生硬截断、语义不完整的广告**。理想状态是 LLM 在生成时就控制在限制内并保持语义完整，截断只作为最后的兜底保险。

## 二、各平台字符限制（本项目配置值）

### Google Ads（响应式搜索广告 RSA）

| 字段 | 字符上限 | 说明 |
|------|---------|------|
| Headline（标题） | **30** | 单条标题上限，一组广告可提供多条 |
| Description（描述） | **90** | 单条描述上限 |
| CTA（行动号召） | 15 | 项目内部约定值 |
| Long Copy（长文案） | 300 | 项目内部约定值 |

- Google 按**字符数**计算（含空格、标点）。
- 中日韩等全角字符通常按显示宽度折算，本项目对 CJK 市场启用"显示宽度"计量。
- 官方参考：[Google Ads 帮助中心 — 关于响应式搜索广告](https://support.google.com/google-ads/answer/7684791)

### Meta / Facebook Ads

| 字段 | 推荐上限 | 说明 |
|------|---------|------|
| Headline（标题） | **40** | 超出在多数版位会被截断 |
| Primary text / Description（正文） | **125** | 官方推荐长度，超出会出现"查看更多"折叠 |
| CTA | 20 | 项目内部约定值 |
| Long Copy | 500 | 项目内部约定值 |

- Meta 的 125 字符是**推荐值**而非硬上限——超出技术上可投放，但 feed 预览会折叠尾部，影响阅读。
- 官方参考：[Meta 广告管理工具 — 广告文案长度建议](https://www.facebook.com/business/help)

### TikTok Ads（竞价 In-Feed 广告）

| 字段 | 字符上限 | 说明 |
|------|---------|------|
| Headline（标题） | **40** | 项目内部约定值 |
| Ad text / Description（广告文案） | **100** | 官方区间约 1–100 字符 |
| CTA | 20 | 多为预设按钮文案 |
| Long Copy | 300 | 项目内部约定值 |

- TikTok In-Feed 广告文案约 **1–100 字符**，且最多显示 **4 行**（含 emoji）。
- Spark Ads（加热原生视频）的文案直接取自原视频，无额外字符限制。
- 官方参考：[TikTok Ads 帮助中心 — 竞价 In-Feed 广告](https://ads.tiktok.com/help/article/tiktok-auction-in-feed-ads)

## 三、三大平台共通的内容政策（高风险红线）

以下是三个平台普遍禁止或严格限制的内容，文案生成时必须规避：

1. **虚假/夸大宣传**：不得使用"100% 保证""稳赚""必中"等无法兑现的绝对化承诺。
2. **虚假紧迫感**：禁止伪造倒计时、虚假库存等误导性催促。
3. **违禁与受限行业**：博彩、金融借贷、医疗保健、加密货币等行业在多数国家需**资质认证**且文案受额外限制。
4. **歧视性内容**：不得基于种族、性别、宗教、性取向等进行歧视或定向排除。
5. **敏感符号与标点**：本项目配置禁止连续感叹号 `!!`、连续问号 `??` 等被视为"垃圾感"的符号。
6. **品牌/商标**：不得未经授权使用第三方品牌名或商标。

> 💡 **博彩/充值类业务特别提示**：Coco 的充值（topup）业务在部分市场可能被归类为受限行业。同一文案在不同国家的过审结果可能不同，这也是"常换常新"功能存在的原因——当平台审核误判时，运营可快速生成合规变体。

## 四、本项目如何落实这些政策

| 政策要求 | 项目实现 |
|---------|---------|
| 字符上限 | `platform_specs/*.json` 按平台 + 文案类型配置上限 |
| 不生硬截断 | 生成阶段提示 LLM 控长；兜底截断按完整单词回退（详见"截断策略"） |
| CJK 显示宽度 | 中日韩市场按显示宽度而非字符数计量 |
| 违禁词规避 | 合规检查器 + 运营自定义 `must_avoid` 双重过滤 |
| 误判应对 | "常换常新"生成合规变体 |
